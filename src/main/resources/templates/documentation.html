📖 MTNG – Technical Documentation

MTNG is a Spring Boot 4.x MVP meeting management platform that enables teachers/administrators to manage students, run live sessions, chat, and store recordings. It uses an orange-branded UI with Thymeleaf server-side rendering and REST JSON APIs.

🏗 Architecture (MVP)

The application follows an MVC pattern with a service layer:

Controller → Service → Repository → H2 In-Memory DB Thymeleaf Templates ← Controller (Model attributes) REST API (/api/**) ← JavaScript fetch() calls
Package Structure
com.Mtng.Mtng ├── config/ DataInitializer, SecurityConfig ├── controller/ DashboardController, StudentApiController, │ MeetingApiController, ChatApiController, RecordingApiController ├── model/ Meeting, Student, ChatMessage, Recording ├── repository/ MeetingRepository, StudentRepository, │ ChatMessageRepository, RecordingRepository └── service/ MeetingService, StudentService, ChatService, RecordingService

🔌 REST API Reference
Student API – /api/students
MethodEndpointDescription
GET /api/students List all students
GET /api/students/{id} Get student by ID
POST /api/students Create new student (JSON body)
PUT /api/students/{id} Update student
DELETE/api/students/{id} Delete student
POST /api/students/{id}/block Toggle block status
POST /api/students/{id}/mute Toggle mute status
Meeting API – /api/meeting
MethodEndpointDescription
GET /api/meeting/active Get active meeting
POST /api/meeting/start Start a meeting
POST /api/meeting/stop Stop active meeting
POST /api/meeting/toggle-recording Toggle full recording
Chat API – /api/chat
MethodEndpointDescription
GET /api/chat/messages Get all messages
POST /api/chat/send Send a message
DELETE/api/chat/clear Clear all messages
Recordings API – /api/recordings
MethodEndpointDescription
GET /api/recordings List all recordings
GET /api/recordings/student/{id} Recordings for student
POST /api/recordings Add recording entry
DELETE/api/recordings/{id} Delete recording
DELETE/api/recordings/clear Clear all recordings
🗄 Data Models
Student
{ id, name, username, password, deviceLock, showRecordings, blocked, muted, status (ONLINE|OFFLINE), createdAt, lastSeen }
Meeting
{ id, title, active, fullRecording, inMeetingCount, onlineCount, startTime, endTime }
ChatMessage
{ id, sender, content, sentAt, meetingId }
Recording
{ id, studentId, studentName, durationSeconds, recordingDate, recordingTime, meetingId }
